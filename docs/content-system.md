# Контентная система

## Как устроен проект

Есть общий React-слой, который ничего не знает о конкретном герое заранее.

Он умеет только:
- строить страницу героя;
- строить нижнюю навигацию по массиву `pages`;
- строить якоря страницы по массиву `sections`;
- рендерить блоки по полю `type`.

Весь контент лежит в `src/content`.

---

## Где лежит контент Пикета

Файл героя:
- `src/content/characters/picket/character.js`

Страницы Пикета:
- `src/content/characters/picket/pages/overview.js`
- `src/content/characters/picket/pages/dice.js`
- `src/content/characters/picket/pages/tools.js`
- `src/content/characters/picket/pages/mechanics.js`

---

## Как добавить новую страницу герою

1. Создай новый файл в папке `pages` нужного героя.

Пример:
- `src/content/characters/picket/pages/positioning.js`

2. Экспортируй объект страницы:

```js
export default {
  id: 'positioning',
  title: 'Позиция',
  sections: [],
}
```

3. Подключи страницу в `src/content/characters/index.js`.

4. Добавь страницу в массив `pages` героя в `character.js`.

Пример:

```js
pages: [
  { id: 'overview', label: 'О герое' },
  { id: 'positioning', label: 'Позиция' },
]
```

После этого кнопка появится в нижнем меню автоматически.

---

## Как поменять порядок страниц в нижнем меню

Открой `character.js` нужного героя и просто переставь элементы внутри массива `pages`.

Именно этот массив является источником правды для нижней навигации.

---

## Как добавить новую секцию на страницу

Открой нужный page-файл и добавь объект в массив `sections`.

Пример:

```js
{
  id: 'mistakes',
  label: 'Ошибки',
  title: 'Частые ошибки',
  blocks: [
    { type: 'paragraph', text: '...' },
  ],
}
```

---

## Как поменять порядок якорей

Порядок якорей берётся из порядка секций в массиве `sections`.

Чтобы поменять порядок якорей в мобильном меню, просто переставь секции местами в page-файле.

---

## Как добавить новый блок в секцию

Открой нужную секцию и добавь новый объект в массив `blocks`.

Пример:

```js
{
  type: 'callout',
  variant: 'tip',
  title: 'Совет',
  text: 'Держи позицию рядом с тем союзником, которого реально надо прикрывать.',
}
```

---

## Какие типы блоков есть сейчас

На старте подключены:
- `paragraph`
- `bulletList`
- `callout`
- `cards`
- `checklist`
- `routeSteps`
- `tracker`
- `statGrid`

Реестр блоков находится здесь:
- `src/components/renderers/blockRegistry.js`

---

## Как создать новый тип блока

1. Создай новый компонент в папке:
- `src/components/blocks/`

Пример:
- `TimelineBlock.jsx`

2. Подключи его в `blockRegistry.js`.

Пример:

```js
import TimelineBlock from '../blocks/TimelineBlock'

export default {
  ...,
  timeline: TimelineBlock,
}
```

3. После этого новый блок можно использовать в контенте:

```js
{
  type: 'timeline',
  items: [...],
}
```

---

## Как добавить нового героя

1. Создай папку:
- `src/content/characters/new-hero/`

2. Создай в ней `character.js`.

3. Создай папку `pages/` и положи туда page-файлы.

4. Подключи героя в `src/content/characters/index.js`.

5. После этого герой автоматически появится на главной странице, если он попал в общий registry.

---

## Главное правило проекта

Общий слой отвечает только за рендер.

Контент героя сам решает:
- какие у него страницы;
- сколько их;
- какие у них секции;
- какие блоки внутри;
- в каком всё порядке.

Нельзя снова превращать систему в жёсткую схему вида:
- overview
- dice
- tools

Это не шаблон фиксированных экранов.
Это файловый конструктор контента.
